The Problem

I wanted to actually understand how a blockchain works under the hood — wallets, consensus, mining, peer-to-peer networking — rather than just reading about the concepts. So I built one from scratch in Java instead of relying on an existing library to handle the hard parts.

My Role

This was a solo project. I implemented the core Block/Blockchain/ BlockConsensus classes, wallets backed by RSA key pairs, a UTXO-style transaction model with inputs and outputs, a Merkle tree for verifying transaction sets, mining logic, and a working socket-based P2P network layer so multiple nodes could actually exchange blocks and messages with each other.

Outcome

The result is a genuinely working peer-to-peer blockchain, not just a single-process simulation. It's the project I point to that shows systems and cryptography fundamentals — distributed consensus, networking, and data structures — rather than framework-driven CRUD work.
